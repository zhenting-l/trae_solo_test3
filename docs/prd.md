# 学术报告识别与总结（Android）PRD

## 目标
用户在听学术报告时，通过拍照或导入幻灯片照片创建一个“报告条目”。系统在用户确认“照片已上传完毕”后，自动调用支持视觉输入的大模型完成：信息抽取（报告人/标题/单位等）、逐页内容结构化、跨页融合总结，并在此基础上做延伸挖掘（相关工作、可能的论文线索、开放问题、与用户领域的连接），最终输出一个可分享的PDF总结文档。

## 核心用户流程
1. 创建条目：输入可选的条目名称/时间/地点（可空）。
2. 采集材料：拍照或从相册选择多张幻灯片图片。
3. 确认上传完成：用户点击“开始分析/生成总结”按钮。
4. 后台分析：逐张解析 → 跨页汇总 → 延伸挖掘 → 生成PDF。
5. 回顾与分享：在历史条目中查看照片、模型输出和PDF；可选择分享照片与/或PDF，若多项则自动打包zip后分享。

## 功能范围
### 必选
- 条目管理：创建/列表/详情。
- 图片采集：系统相机拍摄 + 相册多选导入；本地持久化保存。
- LLM分析：
  - 逐页图片解析（结构化输出：页码、章节/要点、关键公式/图表描述、引用/论文线索）。
  - 报告元信息抽取（报告人、单位、标题、关键词）。
  - 延伸挖掘（相关工作、代表论文、可复现路径、开放问题、建议进一步阅读）。
  - 生成“详版”总结：在综合总结外，包含逐页要点、术语解释、潜在实验/复现建议等。
- 产物：PDF总结文档落盘并关联到条目。

### 附加
- 历史回顾：展示条目内所有照片、模型中间结果、最终PDF。
- 分享：Android Sharesheet；选择分享项；多项则zip打包后分享。
- 低内存：图片按需解码、压缩后上传、避免常驻Bitmap；中间结果与PDF流式写入文件。
- 模型设置：
  - 预设：DeepSeek/智谱(GLM)/阿里通义(Qwen)/自定义。
  - 可配置：baseUrl、apiKey、视觉模型名、文本模型名、超时与最大并发。

## 非功能要求
- 离线策略：无网络时允许拍照/导入与保存；分析任务在网络恢复后可手动重试。
- 隐私：apiKey仅本地保存；日志不打印敏感信息；分享时明确用户选择内容。
- 性能：分析任务后台执行，不阻塞UI；失败可重试；可显示进度与最近错误。

## 关键页面
- 条目列表页：新建、搜索/排序（按时间）、状态标签（未分析/分析中/成功/失败）。
- 条目详情页：
  - 图片网格 + 添加按钮（相机/相册）
  - “照片上传完成/开始分析”按钮
  - 模型输出：元信息、逐页解析摘要、最终总结入口
  - 分享按钮：进入选择分享内容
- 设置页：模型供应商预设、baseUrl/apiKey、视觉/文本模型选择、测试连通性（可选）。
