## Summary of Slides（Android）用户手册

版本：1.2.0（code 6）

### 1. 应用用途

把学术报告/课程/组会的幻灯片或论文 PDF 导入手机后，调用大模型进行解析与总结，生成可阅读/可分享的 Markdown 与 HTML 总结，并支持多文件一键打包分享。

### 2. 名词说明

- 条目：一次报告/一次论文的一组材料（图片与/或 PDF）及其分析结果。
- 视觉模型：用于识别图片内容（幻灯片拍照/截图、或 PDF 回退渲染页）的模型。
- 通用模型：用于融合多页解析结果、生成最终总结（Markdown）的模型。
- 分析任务：后台 WorkManager 执行的分析流程；同一时间全局最多运行 1 个任务（适配接口频率限制）。

### 3. 首次使用（必须）

1) 进入：设置 → 模型调用设置

2) 配置模型信息

- baseUrl：你的模型服务地址
- apiKey：你的密钥
- 视觉模型：用于图片识别
- 通用模型：用于总结生成

3) 可选：联网补充论文信息

- 开启后，论文 PDF 的总结会尝试使用公开学术数据库补全元数据（Crossref / arXiv），用于更准确地补齐标题、作者、关键词等字段。

### 4. 日常使用流程（推荐）

#### 4.1 创建条目

1) 在条目列表页点击“新建”
2) 进入条目详情页

#### 4.2 导入材料

条目详情页支持 3 种输入：

1) 连续拍照

- 点击“连续拍照”
- 每拍完一张会自动再次打开相机
- 在相机界面取消/返回即可结束连续拍照

2) 相册导入

- 点击“相册导入”，批量选择多张图片

3) 上传 PDF

- 点击“上传PDF”
- 可在“查看已上传PDF”中检查已添加的 PDF 列表

注意：

- 单个条目可同时包含图片与 PDF；系统会优先基于图片逐页解析，并在可用时融合 PDF 抽取的 Markdown。

#### 4.3 选择分析方式

分析开始前你可以设置：

1) 附加提示词（可选）

- 点击“附加提示词”按钮
- 输入你希望模型额外遵循的要求（例如更侧重公式推导、希望输出更偏论文口吻等）
- 该提示词会附加到图片解析与总结提示词中

2) 图片输入模式

- 逐张输入：每张图片单独调用视觉模型解析（更稳，但可能更慢）
- 批量输入：一次请求传入多张图片让模型返回 JSON 数组（更快，但对模型输出格式要求更高）

系统保护：

- 若批量输出无法解析或返回数量不匹配，会自动回退为逐张解析，避免“逐页解析为空”的异常总结。

#### 4.4 开始分析 / 中断任务

1) 第一次总结（无历史总结时）

- 直接点击“开始分析并生成总结”

2) 已有历史总结时

- 点击“重新开始分析总结”
- 弹窗选择：
  - 重新总结：对全部图片重新解析并生成新的总结
  - 增量总结：仅解析新增图片，并融合上次总结生成新总结

3) 中断任务

- 当状态为 QUEUED / PROCESSING 时，按钮会显示“中断任务”
- 点击即可取消后台任务

### 5. 输出结果说明

分析完成后会生成：

1) Markdown 总结（md）

- 在条目详情页点击“查看总结（Markdown）”
- 内容包含报告信息、整体摘要、逐页要点、术语解释、延伸建议等（取决于模型输出）

2) HTML 总结（html）

- 用于分享或在系统浏览器中打开

3) 图片文件命名与展示

- 分析完成后，图片会按“页码-简短说明.jpg”形式重命名，并更新 App 内展示名称
- 简短说明优先取自逐页解析 JSON 的 section / main_points（尽量几个字）

### 6. 分享流程

1) 在条目详情页右上角点击“分享”
2) 在分享列表中按分类勾选：

- MD 总结文档
- HTML 文档
- 用户上传的文件（PDF 等）
- 图片（会显示为“页码-简短说明.jpg”）

3) 点击分享按钮

- 单文件：直接分享
- 多文件：自动打包为 zip 后分享

### 7. 常见问题与排查

#### 7.1 “账号频率限制/同一时间只能跑一个任务”

- 应用已强制全局同一时间最多 1 个分析任务
- 若你同时在多个条目点了开始分析，后发起的会提示等待或中断

#### 7.2 批量输入导致结果异常

- 批量输出要求模型返回 JSON 数组且数量严格匹配图片数
- 若模型不支持/输出不规范，应用会自动回退到逐张解析
- 建议遇到异常时切换到“逐张输入”

#### 7.3 Markdown 公式渲染失败

应用对 Markdown 中的 LaTeX 公式做了规范化与纠错，但仍建议：

- 块级公式用独立行的 $$ 包裹（$$\n...\n$$）
- 行内公式 $...$ 不要跨行
- 上下标使用 ^/_ 并配合大括号（x_{k}, x^{k}）

如果仍有渲染问题，优先检查是否存在未闭合的 $ / $$、或公式里混入了全角标点。

#### 7.4 论文 PDF 总结质量不稳定

论文 PDF 版式复杂且内容长，应用采取：

- 优先使用 PDF 版式解析抽取 Markdown
- 对较长/论文结构明显的 PDF 启用分块总结再融合
- 若 PDF 解析失败，会回退为逐页渲染成图片并用视觉模型解析（效果取决于渲染质量与清晰度）

### 8. 最佳实践建议

- 幻灯片拍照尽量正对、避免反光、保证文字清晰
- 幻灯片较多时优先“逐张输入”保证稳定；接口稳定后再尝试“批量输入”
- 每次新增少量页后，优先用“增量总结”节省时间与成本
- 需要更强的“报告信息/关键词补齐”时，开启“联网补充论文信息”
